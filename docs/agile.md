## ¿Qué es Agile?

Agile es una metodología de trabajo basada en el desarrollo iterativo e incremental de software. Su objetivo es entregar valor al usuario de forma continua mediante pequeñas mejoras frecuentes, permitiendo adaptarse rápidamente a cambios en los requisitos.

## ¿Qué es Scrum?

Scrum es un marco de trabajo Agile que organiza el desarrollo en ciclos cortos llamados sprints.

### Roles principales

* Product Owner: define prioridades y requisitos.
* Scrum Master: facilita el proceso y elimina obstáculos.
* Development Team: desarrolla el producto.

### Conceptos principales

* Product Backlog
* Sprint Backlog
* Sprint
* Sprint Review
* Sprint Retrospective

## ¿Qué es Kanban?

Kanban es un sistema visual para gestionar tareas mediante tableros. Las tareas se representan mediante tarjetas que avanzan por diferentes estados.

Ejemplo:

* Backlog
* Todo
* In Progress
* Review
* Done

## Diferencias entre Scrum y Kanban

### Scrum

* Trabaja por sprints.
* Tiene roles definidos.
* Planificación periódica.

### Kanban

* Flujo continuo.
* No requiere sprints.
* Más flexible para cambios inmediatos.

## ¿Cuándo usar cada uno?

Scrum es recomendable cuando existe un proyecto con objetivos claros y entregas periódicas.

Kanban es útil cuando las tareas cambian constantemente y se necesita flexibilidad para reorganizar prioridades.
